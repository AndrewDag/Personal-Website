Summary

In my time with Ford, I worked on a team that designed an automated tool to streamline the creation of Terraform infrastructure for various Google Cloud Services as a method of canary testing each service.

The tool accesses GitHub and analyzes JSON files to pull nessecary Terraform files from remote repositories. The tool then places them into a local environment where Terraform infrastrcture can be built. It can even make changes to individual values in the Terraform files if it was deemed nessecary to do so.

Utilizing Python in conjunction with Flask, a localized application is created and it allows for seamless connection between the tool's frontend and backend components.
